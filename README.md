# 키친포스

## 요구 사항

### 상품

- 상품을 등록할 수 있다.
- 상품의 가격이 올바르지 않으면 등록할 수 없다.
    - 상품의 가격은 0원 이상이어야 한다.
- 상품의 이름이 올바르지 않으면 등록할 수 없다.
    - 상품의 이름에는 욕설이 포함될 수 없다.
- 상품의 가격을 변경할 수 있다.
- 상품의 가격이 올바르지 않으면 변경할 수 없다.
    - 상품의 가격은 0원 이상이어야 한다.
- 상품의 가격이 변경될 때 메뉴의 가격이 메뉴에 속한 상품 금액의 합보다 크면 메뉴가 숨겨진다.
- 상품의 목록을 조회할 수 있다.

### 메뉴 그룹

- 메뉴 그룹을 등록할 수 있다.
- 메뉴 그룹의 이름이 올바르지 않으면 등록할 수 없다.
    - 메뉴 그룹의 이름은 비워 둘 수 없다.
- 메뉴 그룹의 목록을 조회할 수 있다.

### 메뉴

- 1 개 이상의 등록된 상품으로 메뉴를 등록할 수 있다.
- 상품이 없으면 등록할 수 없다.
- 메뉴에 속한 상품의 수량은 0 이상이어야 한다.
- 메뉴의 가격이 올바르지 않으면 등록할 수 없다.
    - 메뉴의 가격은 0원 이상이어야 한다.
- 메뉴에 속한 상품 금액의 합은 메뉴의 가격보다 크거나 같아야 한다.
- 메뉴는 특정 메뉴 그룹에 속해야 한다.
- 메뉴의 이름이 올바르지 않으면 등록할 수 없다.
    - 메뉴의 이름에는 욕설가 포함될 수 없다.
- 메뉴의 가격을 변경할 수 있다.
- 메뉴의 가격이 올바르지 않으면 변경할 수 없다.
    - 메뉴의 가격은 0원 이상이어야 한다.
- 메뉴에 속한 상품 금액의 합은 메뉴의 가격보다 크거나 같아야 한다.
- 메뉴를 노출할 수 있다.
- 메뉴의 가격이 메뉴에 속한 상품 금액의 합보다 높을 경우 메뉴를 노출할 수 없다.
- 메뉴를 숨길 수 있다.
- 메뉴의 목록을 조회할 수 있다.

### 테이블

- 테이블을 등록할 수 있다.
- 테이블의 이름이 올바르지 않으면 등록할 수 없다.
    - 테이블의 이름은 비워 둘 수 없다.
- 빈 테이블을 해지할 수 있다.
- 빈 테이블로 설정할 수 있다.
- 완료되지 않은 주문이 있는 테이블은 빈 테이블로 설정할 수 없다.
- 방문한 손님 수를 변경할 수 있다.
- 방문한 손님 수가 올바르지 않으면 변경할 수 없다.
    - 방문한 손님 수는 0 이상이어야 한다.
- 빈 테이블은 방문한 손님 수를 변경할 수 없다.
- 테이블의 목록을 조회할 수 있다.

### 주문

- 1개 이상의 등록된 메뉴로 배달 주문을 등록할 수 있다.
- 1개 이상의 등록된 메뉴로 포장 주문을 등록할 수 있다.
- 1개 이상의 등록된 메뉴로 매장 주문을 등록할 수 있다.
- 주문 유형이 올바르지 않으면 등록할 수 없다.
- 메뉴가 없으면 등록할 수 없다.
- 매장 주문은 주문 항목의 수량이 0 미만일 수 있다.
- 매장 주문을 제외한 주문의 경우 주문 항목의 수량은 0 이상이어야 한다.
- 배달 주소가 올바르지 않으면 배달 주문을 등록할 수 없다.
    - 배달 주소는 비워 둘 수 없다.
- 빈 테이블에는 매장 주문을 등록할 수 없다.
- 숨겨진 메뉴는 주문할 수 없다.
- 주문한 메뉴의 가격은 실제 메뉴 가격과 일치해야 한다.
- 주문을 접수한다.
- 접수 대기 중인 주문만 접수할 수 있다.
- 배달 주문을 접수되면 배달 대행사를 호출한다.
- 주문을 서빙한다.
- 접수된 주문만 서빙할 수 있다.
- 주문을 배달한다.
- 배달 주문만 배달할 수 있다.
- 서빙된 주문만 배달할 수 있다.
- 주문을 배달 완료한다.
- 배달 중인 주문만 배달 완료할 수 있다.
- 주문을 완료한다.
- 배달 주문의 경우 배달 완료된 주문만 완료할 수 있다.
- 포장 및 매장 주문의 경우 서빙된 주문만 완료할 수 있다.
- 테이블의 모든 매장 주문이 완료되면 빈 테이블로 설정한다.
- 완료되지 않은 매장 주문이 있는 테이블은 빈 테이블로 설정하지 않는다.
- 주문 목록을 조회할 수 있다.

## 용어 사전

| --- | --- | --- |

| 메뉴 | Menu | 주문을 통해 주문자에게 판매될 수 있는 단위, 하나 이상의 상품을 포함해야 한다 |

| 노출된 메뉴 | display | 주문자가 주문할 수 있는 메뉴 |
| 숨겨진 메뉴 | hide | 주문자가 주문할 수 없는 메뉴 |
| 방문한 손님 | guests | 매장에 방문해서 주문 하려는 주문자 |
| 테이블 | OrderTable | 매장 주문을 하려는 손님들이 차지하는 있는 공간 |
| 빈 테이블 | empty | 테이블을 차지한 방문한 손님이 0명인 상태 |
| 주문자 | Orderer | 주문을 요청해서 음식을 구입하려는 사람 |
| 주문 | orders | 주문자가 가격을 지불하고 주문 항목(메뉴와 수량)을 요청하는 것 |
| 주문 항목| OrderLineItem | 주문자가 받고자 하는 각 메뉴와 수량 |
| 주문 유형 | OrderType | 주문자가 주문을 등록하는 방법이며 {배달 주문, 매장 주문, 포장 주문} 이 있음 |
| 배달 주문 | DELIVERY | 주문자가 배달주소에서 주문 항목을 수령하는 주문유형 |
| 매장 주문 | EAT_IN | 주문자가 매장에 방문하여 주문하고, 수령하는 주문유형 |
| 포장 주문  | TAKEOUT | 주문자가 매장에 미리 주문을 요청하고, 추후 방문하여 주문 항목을 수령하는 주문유형 |
| 배달원 | Rider | 주문 항목을 주문자가 지정한 배달주소로 옮겨주는 주체 |
| 배달 주소 | DeliveryAddress | 주문자가 주문 항목의 수령을 원하는 장소 |
| 주문상태  | OrderStatus | 주문이 준비된 상태 |

| 서빙 | SERVED   | 주문 항목의 조리가 완료된 상태 <br/> 주문 유형이 포장 주문, 매장 주문인경우 주문자에게 제공하고, 배달이면 배달원 도착을 기다린다 |

## 모델링

###상품(Product)
####속성
- `상품`은 `욕설(Profanity)`이 포함될 수 없는 `상품명(ProductName)`을 가진다
- `상품`은 `상품가격정책`을 따르는 `상품가격(ProductPrice)`을 가진다

####행위
- `ProductService`는 새로운 `상품`을 등록할 수 있다
  - 상품 속성에 부합하지 않는 `상품`은 등록할 수 없다
- `ProductService`는 등록된 `상품`의 `상품가격`을 변경할 수 있다
  - `상품가격정책`을 위반하는 `상품가격`으로는 변경할 수 없다
  - 가격변경으로 인해 `상품`이 속한 `메뉴`가 `메뉴가격정책`을 위반하게 되는 경우 `메뉴`를 숨김처리 한다
- `ProductService`는 모든 `상품`을 조회할 수 있다

###메뉴(Menu)
####속성
- `메뉴`는 `욕설(Profanity)`이 포함될 수 없는 `메뉴명(MenuName)`을 가진다
- `메뉴`는 `메뉴가격정책`을 따르는 `메뉴가격(MenuPrice)`을 가진다
- `메뉴`는 `노출상태(displayed)`로 `노출(display)`, `숨김(hide)`중 하나를 가진다
- `메뉴`는 하나의 `메뉴그룹`에 속해야 한다
- `메뉴`는 하나 이상의 `메뉴 항목(MenuProduct)`을 가진다
  - `메뉴 항목`은 `상품(Product)`과 `수량(quantity)`로 구성된다
####행위
- `MenuService`는 새로운 `메뉴`를 등록할 수 있다
  - 메뉴 속성에 부합하지 않는 `메뉴`는 등록할 수 없다
- `MenuService`는 등록된 `메뉴`의 `메뉴가격`을 변경할 수 있다
  - `메뉴가격정책`을 위반하는 `메뉴가격`으로는 변경할 수 없다
- `MenuService`는 등록된 `메뉴`의 `노출상태`를 `노출(display)`로 변경할 수 있다
  - `메뉴가격정책`을 위반하는 `메뉴`는 `노출`로 변경할 수 없다
- `MenuService`는 등록된 `메뉴`의 `노출상태`를 `숨김(hide)`로 변경할 수 있다
- `MenuService`는 모든 `메뉴`를 조회할 수 있다
 

###메뉴그룹(MenuGroup)
####속성
- `메뉴그룹`은 한글자 이상의 `메뉴그룹명(MenuGroupName)`을 가진다
-  `메뉴그룹`은 반드시 하나 이상의 `메뉴`를 가진다
####행위
- `MenuGroupService`는 새로운 `메뉴그룹`을 등록할 수 있다
  - 메뉴그룹 속성에 부합하지 않는 `메뉴그룹`은 등록할 수 없다
- `MenuGroupService`는 모든 `메뉴그룹`을 조회할 수 있다


###주문(Order)
####속성
- `주문`은 `배달 주문(DELIVERY)`, `매장 주문(EAT_IN)`, `포장 주문(TAKEOUT)`중 하나의 `주문 유형(OrderType)`을 가진다
- `주문`은 `주문 상태(OrderStatus)`를 가진다
  - `주문 상태`는 `접수대기(WAITING)`, `접수(ACCEPTED)`, `서빙(SERVED)`, `배달중(DELIVERING)`, `배달완료(DELIVERED)`, `주문완료(COMPLETED)`중 하나 이다
- `주문`은 `주문 시간(OrderDateTime)`을 가진다
- `주문`은 `주문 유형`이 `배달 주문`인 경우 `배달 주소(deliveryAddress)`를 가진다
- `주문`은 하나 이상의 `주문 항목(OrderLineItem)`을 가진다
  - `주문항목`은 `메뉴(Menu)`와 `수량(quantity`), `가격(메뉴의 가격 * 수량)`으로 구성된다
####행위
- `OrderService`는 새로운 `주문`을 등록할 수 있다
  - 주문 속성에 부합하지 않는 `주문`은 등록할 수 없다
  - `배달 주문`, `포장 주문`의 경우 각 `주문 항목(OrderLineItem`)의 `수량(quantity)`은 0 이상이어야 한다
  - `매장주문`의 경우 각 `주문 항목`의 `수량`은 0 미만일 수 있고, `빈 테이블`에는 `주문`을 등록할 수 없다
  - 주문하려는 `메뉴`의 가격은 등록된 `메뉴`의 `가격`과 일치해야 한다
  - 숨겨진 `메뉴`는 주문할 수 없다
  - `주문` 등록 후 `주문 상태`는 `접수 대기` 이다
- `OrderService`는 `주문 상태`를 `접수`로 변경할 수 있다
  - 기존 `주문 상태`가 `주문 대기`인 경우만 가능하다
  - `주문 유형`이 `배달주문`인 경우 `배달원`을 호출한다
- `OrderService`는 `주문 상태`를 `서빙`으로 변경할 수 있다
  - 기존 `주문 상태`가 `접수`인 경우만 가능하다
- `OrderService`는 `주문 상태`를 `배달중`으로 변경할 수 있다
  - `주문 유형`이 `배달 주문`이고, 기존 `주문 상태`가 `서빙`인 경우만 가능하다
- `OrderService`는 `주문 상태`를 `배달완료`로 변경할 수 있다
  - 기존 `주문 상태`가 `배달중`인 경우만 가능하다
- `OrderService`는 `주문 상태`를 `주문완료`로 변경할 수 있다
  - `주문 유형`이 `배달 주문`인 경우 기존 `주문 상태`가 `배달 완료`인 경우만 가능 하다
  - `주문 유형`이 `포장 주문`, `매장 주문`인 경우 기존 `주문 상태`가 `서빙`인 경우만 가능하다
    - `주문 상태` 변경 후 `테이블`을 `빈 테이블`로 설정한다
  - `주문 유형`이 `포장 주문`경우 기존 `주문 상태`가 `서빙`인 경우만 가능하다
- `OrderService`는 모든 `주문`을 조회할 수 있다


###테이블(OrderTable)
####속성
- `테이블`은 한글자 이상의 `테이블명(OrderTableName)`을 가진다
- `테이블`은 `빈 테이블 여부(empty)`를 가진다
- `테이블`은 0명 이상의 `착석인원(numberOfGuests)`을 가진다
####행위
- `OrderTableService`는 새로운 `테이블`을 등록할 수 있다
  - 신규 테이블은 속성에 부합해야 하고 등록 후 `빈 테이블 여부`는 `빈 테이블`로, `착석인원`은 0명으로 한다
- `OrderTableService`는 `테이블`을 `비어있지 않은 테이블`로 변경할 수 있다
- `OrderTableService`는 `주문`이 `주문 종결`된 `테이블`에 한하여 `빈 테이블`로 변경할 수 있다
  - 변경 이후 `착석인원`은 0명으로 설정한다
- `OrderTableService`는 `비어있지 않은 테이블`의 `착석 인원`을 변경할 수 있다
- `OrderTableService`는 `모든 테이블`을 조회할 수 있다
